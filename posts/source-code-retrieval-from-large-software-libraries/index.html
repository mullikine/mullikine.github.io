Conclusions

Aautomatic BL that takes into account SCP and term ordering relationships in a code base for improved retrieval accuracy.

Retrieval framework has a fuzzy/combinatorial matching mechanism between the term blocks of the queries (multiple words) and the source code.

MRF that captures both the positional and the order attributes of the terms in source files.

With MRF model, the proposed QC approach effectively exploits the different types of structural information that is frequently included in bug reports. We showed that the structural elements in bug reports – particularly stack traces – contain vital information that is not well represented vis-à-vis the source code with the widely used BOW assumption. Overall, on the basis of retrieval accuracies, it is clear that one can obtain significantly higher bug localization accuracies when MRF modeling and QR are included in a retrieval framework compared with all other state-of-the-art approaches, even including those that take prior development history into account.

Definitions

  1. P1 The probability of having tf occurrences of the term in the document by pure chance. As this decreases, the information content of the document (vis-a-vis/in relation with) the term increases.

  2. P2 (1 - P2) is related to the risk of choosing the query term as a discriminative term and works as a normalization factor.

    Can be used as the probability of having one more occurrence of the term in the document, which leads to penalizting the high frequency terms during retrieval.

  3. BoWa :: Each word is sampled independantly from the rest of the words in the document.

  4. BoW / VSM :: A simplifying representation. Each d is a vector of Wf.

  5. LSI :: An early retrieval method. Assign greater importance to terms that frequently co-occur in source files. Can use to expand a given initial query that consists of a single query term initially.

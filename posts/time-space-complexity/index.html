∊

T(n) ∊ O(f(n)) means that T(n) doesn’t grow faster than f(n).

Constant

An algorithm with T(n) ∊ O(1) is said to have constant time complexity.

Linear

An algorithm with T(n) ∊ O(n) is said to have linear time complexity.

Quadratic

The second algorithm in the Time complexity article had time complexity T(n) = n2/2 - n/2. With Big O notation, this becomes T(n) ∊ O(n2), and we say that the algorithm has quadratic time complexity.

How to analyse

Space complexity

1
2
3

vector<int> myVec(n);
for(int i = 0; i < n; i++)
    cin >> myVec[i];

In the above example, we are creating a vector of size n. So the space complexity of the above code is in the order of “n” i.e. if n will increase, the space requirement will also increase accordingly.

Even when you are creating a variable then you need some space for your algorithm to run. All the space required for the algorithm is collectively called the Space Complexity of the algorithm.

Time complexity

The time complexity is the number of operations an algorithm performs to complete its task with respect to input size.

Count your steps

Calculated by counting elementary operations.

What’s the running time of the following algorithm?

// Compute the maximum element in the array a.
Algorithm max(a):
    max ← a[0]
    for i = 1 to len(a)-1
        if a[i] > max
            max ← a[i]
    return max
  • Steps

    • choose an elementary operation, which the algorithm performs repeatedly,

      Use comparisons since they dominate all other operations. Specifically, choose the a[i] > max comparison as the one used to measure.

    • define the time complexity T(n) as the number of such operations the algorithm performs given an array of length n.

      The time complexity, measured in the number of comparisons, then becomes T(n) = n - 1.

      Because the loop always runs n - 1 times.

  • The elementary operation must have 2 properties:

    • There can’t be any other operations that are performed more frequently as the size of the input grows.

    • The time to execute an elementary operation must be constant: It mustn’t increase as the size of the input grows. This is known as unit cost.

Best-case time complexity

We must know the case that causes the minimum number of operations to be executed.

Worst-case time complexity

We must know the case that causes the maximum number of operations to be executed.

// Tell whether the array a contains x.
Algorithm contains(a, x):
    for i = 0 to len(a)-1
        if x == a[i]
            return true
    return false

The value might not be found in the list.

W(n) = n.

Gives an upper bound on time requirements and is often easy to compute. The drawback is that it’s often overly pessimistic.

Average-case time complexity

We calculate the running time for all possible inputs, sum all the calculated values and divide the sum by the total number of inputs. We must know (or predict) distribution of cases.

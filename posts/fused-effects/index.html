Install fused-effects

stack install fused-effects

Slides

fused-effects is

  • a library for building Haskell programs
  • an effect system providing fused, scoped, higher-order algebraic effects.
  • performant, production-quality software suitable for industrial use.

Today, we’ll…

  • describe the status quo of building Haskell programs
  • explore how this status quo can be limiting
  • show how fused-effects can overcome these limitations
  • pinpoint fused-effects’s position in the design space

How are we to build programs in Haskell?

  • The world is all that is the case.
  • A computation is any action taken by a programmable device.
  • An environment is the set of all past and present worlds that a computation can observe.
  • An effect is a computation that depends on or changes the program’s environment.
  • We run programs to produce effects in the world around us.

Effects

  • input/output
  • state changes
  • thrown exceptions
  • nondeterminism
  • resource management
  • tracing
  • failure
  • configuration
  • randomness

Slides

  • Programming languages provide you with the ability to express effects.
  • Haskell tracks the presence and scope of effects with types.
  • Before I can write Haskell code, I need to think about the effects I need.

IO is a domain-specific language, embedded in Haskell, for communicating with the operating system.

Quote

Despite Haskell being a functional programming language it’s really big on data structures and most notably data structures that compose

  • that conform to a certain interface and that’s the dreaded M word, ‘monads’.

Monads

monads are a unified interface for a great many useful effects

helloWorld :: IO ()
helloWorld = do
putStrLn "What's your name?"

name <- getLine

putStrLn ("Hello, " ++ name ++ "!")

Monads allow allow us using the do notation and the bind arrow operation to write imperative code in a functional setting but this index generalizes beyond just IO.

pythagTriples :: [(Int, Int, Int)]

pythagTriples = do
a <- [0..10]
b <- [0..10]
c <- [0..10]
guard (a^2 + b^2 == c^2)

return (a, b, c)

Explanation

--stream turns the json into a list of the paths to the values.

And then there’s kind of an end marker for every array or object.

1

echo '[{"foo":"bar"},{"foo":"baz"}]' | jq --stream -c .
[[0,"foo"],"bar"]
[[0,"foo"]]
[[1,"foo"],"baz"]
[[1,"foo"]]
[[1]]

The addresses:

1
2

[[0,"foo"],"bar"]
[[1,"foo"],"baz"]

The end markers:

1
2
3

[[0,"foo"]]
[[1,"foo"]]
[[1]]

Here it is after the parent list in the paths is deleted by truncate_stream.

The 0 1 are deleted.

There’s no more array, so there’s no more end of array marker [[1]].

You could put 0 instead of 1 before |truncate_stream to see it before the change.

1

echo '[{=foo":"bar"},{"foo":"baz"}]' | jq -cn --stream '1|truncate_stream(inputs)'
[["foo"],"bar"]
[["foo"]]
[["foo"],"baz"]
[["foo"]]

And then fromstream() turns it back into json. Now there’s no longer the square brackets for an array on the outside.

It’s just two objects.

1

echo '[{"foo":"bar"},{"foo":"baz"}]' | jq -cn --stream 'fromstream(1|truncate_stream(inputs))'
{"foo":"bar"}
{"foo":"baz"}

reduce

1

[3,2,1]

1

reduce .[] as $item (0; . + $item)
6

For each result that .[] produces, . + $item is run to accumulate a running total, starting from 0. In this example, .[] produces the results 3, 2, and 1, so the effect is similar to running something like this:

1
2
3

0 | (3 as $item | . + $item) |
    (2 as $item | . + $item) |
    (1 as $item | . + $item)

foreach

https://stedolan.github.io/jq/manual/#foreach/h3/code

The form is foreach EXP as $var (INIT; UPDATE; EXTRACT).

Like reduce, INIT is evaluated once to produce a state value, then each output of EXP is bound to $var, UPDATE is evaluated for each output of EXP with the current state and with $var visible.

Each value output by UPDATE replaces the previous state.

Finally, EXTRACT is evaluated for each new state to extract an output of foreach.

This is mostly useful only for constructing reduce- and limit-like functions. But it is much more general, as it allows for partial reductions (see the example below).

[
# .[] produces outputs
# $item is the name of the current input
foreach .[] as $item (
  # Initial state/value
  [[],[]];

  # Update - replace previous state/value
  # When it's null, create a new list that contains a new empty list as a sibling to the first element.
  # In the else clause, the first element is a list that grows with each new item.
  if $item == null then [[],.[0]] else [(.[0] + [$item]),[]] end;

  # Extract -- produce
  if $item == null then .[1] else empty end)
]

    1,
    2,
    3,
    4
  ],
  [
    "a",
    "b"
  ],
  [
    "yo"
  ]
]

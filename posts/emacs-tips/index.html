Help key – interpretted by emacs, not tmux

This is because you would have to modify both terminfo and tmux c source to get the help key this way.

It was simply easier to intercept the raw ansi in emacs and translate it.

(define-key key-translation-map (kbd "C-M-]") (kbd "<help>"))
kb tr
<C-M-]> <help>

Multiple actions as single undo entry

https://www.reddit.com/r/emacs/comments/6scoyy/multiple%5Factions%5Fas%5Fsingle%5Fundo%5Fentry/

(defun org-babel-raise ()
  (interactive)
  (evil-with-single-undo
    (ekm "C-c '")
    (ekm "C-x h")
    (ekm "C-x h")
    (ekm "M-w")
    (ekm "C-c '")
    (call-interactively 'org-babel-mark-block)
    (ekm "<up>")
    (ekm "C-x C-x")
    (ekm "<down>")
    (ekm "C-y")
    (ekm "C-x C-x")))

High precedence key binding

If you want a highest precedence global map, general.el has an override mode.

The examples in the readme are for evil, but it works for emacs (without evil) as well.

(general-def :keymaps 'override "C-x C-f" 'my-find-file-command)

This is a global map. What if I want a minor / maor jode map to override?

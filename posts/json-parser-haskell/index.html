Define the AST

JsonObject (Map String)

Ideally it has to be a map that takes a string as a key and returns a JSON value) but Map is a 3rd party dependency, unfortunately.

In the standard library of haskell there are no maps at all.

They are located in a package called containers, but we are not going to use that because we’re not using any 3rd party dependencies.

Derive interfaces

We have derived our AST type but I also want to derive interfaces like Show and Eq (i.e. equal).

Show is needed so you can take, for example, JsonNumber, which is a JsonValue, type it in the REPL and the REPL will print it back.

Once it implements show, you can print it.

thoughts
if you want to make something able to be printed, you have to work for it. That means you have to do something and in this case that something is the 4 letters “show”.

We also implement Eq so that we can compare the values of this JSON stuff.

Define the parser

Create separate smaller parsers first. Then combine those parsers into a bigger parser that can read the whole JSON. This is similar to recursive descent.

A parser takes a string and returns the value we want to parse.

In general the parser should be able to return anything we want to parse, but since the consituent parsers individually can parse only one specific thing we don’t want to discard the rest of the string. Therefore, we want to return both the thing we want and the remainder of the string.

-- This is even better
newtype Parser a = MkParser
  { runParser :: String -> Maybe (String, a)
  }

The lesson here is that a parser isn’t just a function from string to a.

Fortunately, managing complexity is a very strong characteristic of haskell.

We want to reason about a parser as a type rather than a function

A parser is characterised by the thing it parses.

Instead, create a newtype.

1

newtype Parser a = MkParser { String -> Maybe (String, a)

A newtype is supposed to be a type that has only 1 data constructor that wraps only 1 single type.

We use a record {} for this particular situation.

A record in haskell is like a datastructure struct in C.

This record contains only one field and the type of that field is this function: String -> Maybe (String, a).

  }

Record syntax was used because it automatically creates value constructors.

You put a hole somewhere where an expression is expected.

charP :: Char -> Parser Char
charP x = MkParser f
  where
    f (y:ys)
      | y == x = Just (ys, x)
      | otherwise = Nothing
    f [] = Nothing

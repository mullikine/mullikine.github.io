fzf vs helm, ivy and completing-read

Sep 21, 2019
3 min read
emacselisp

Helm, ivy and completing-read are 3 emacs-lisp alternatives to fzf.

They are all a little different to hack on / configure though.

helm

Select from a list

There are different ways to select from a list.

(defun fz-must-match (list &optional prompt)
  (setq prompt (or prompt ":"))
  (helm-comp-read prompt list :must-match t))

;; This is very useful
(defun fz-default-return-query (list &optional prompt)
  (setq prompt (or prompt ":"))
  (helm-comp-read prompt list :must-match 'confirm))

;; But I don't like confirm things
(defun fz-default-return-query (list &optional prompt)
  (setq prompt (or prompt ":"))
  (helm-comp-read prompt list :must-match 'nil))

Getting the list of options in another window

tv takes a string and opens a window with it.

The key point here is (helm-marked-candidates-strings :all-sources t).

Make your own tv function.

(defun helm-copy-to-tv ()
  "Copy selection or marked candidates to `helm-current-buffer'.
Note that the real values of candidates are copied and not the
display values."
  (interactive)

  (helm-mark-all t)
  (tv (helm-marked-candidates-strings :all-sources t) :b-nowait t :b-quiet t)
  )

(define-key helm-map (kbd "C-c o")      'helm-copy-to-tv)

Copy the currently selected item

(defmacro helm-marked-candidates-strings (&rest body)
  ""
  `(my/join (helm-marked-candidates ,@body) "\n"))

(defun helm-copy-selection-to-clipboard ()
  "Copy selection or marked candidates to `helm-current-buffer'.
Note that the real values of candidates are copied and not the
display values."
  (interactive)

  (my/copy (helm-marked-candidates-strings)))

(define-key helm-map (kbd "M-c") #'helm-copy-selection-to-clipboard)

ivy

Copy the currently selected item

(defun ivy-copy-selection ()
  "Copy the selected candidate as a string."
  (interactive)
  (if (ivy--prompt-selected-p)
      (ivy-immediate-done)
    (setq ivy-current-prefix-arg current-prefix-arg)
    (cond ((or (> ivy--length 0)
               ;; the action from `ivy-dispatching-done' may not need a
               ;; candidate at all
               (eq this-command 'ivy-dispatching-done))
           (my/copy (ivy-current-string) t)
           )
          ((memq (ivy-state-collection ivy-last)
                 '(read-file-name-internal internal-complete-buffer))
           (if (or (not (eq confirm-nonexistent-file-or-buffer t))
                   (equal " (confirm)" ivy--prompt-extra))
               (ivy--done (ivy-current-string))
             (setq ivy--prompt-extra " (confirm)")
             (my/copy (ivy--done (ivy-current-string)) t)
             (ivy--exhibit)))
          ((memq (ivy-state-require-match ivy-last)
                 '(nil confirm confirm-after-completion))
           (ivy--done (ivy-current-string)))
          (t
           (setq ivy--prompt-extra " (match required)")
           (my/copy (ivy-current-string) t)
           (ivy--exhibit)))))

Getting the list of options in another window

(defun ivy-tv-filtered-candidates ()
  (interactive)
  (tvd (list2str (ivy-filtered-candidates))))

Set key bindings for above

(use-package ivy
       :defer 0.1
       :diminish
       :bind (("M-D" . send-m-del)
              ("M-c" . ivy-copy-selection)
              ("C-c o" . ivy-tv-filtered-candidates)
              )
       :custom
       (ivy-count-format "(%d/%d) ")
       (ivy-display-style 'fancy)
       (ivy-use-virtual-buffers t)
       :config (ivy-mode))

completing-read

helm and ivy use completing-read under the hood.

Basic example

(completing-read "test: " '(a b c d) nil t)
(completing-read "test: " '(a b c d) nil 'confirm)

Useful functions

These are different ways to select from a list.

(defun fz-must-match (list &optional prompt)
  (setq prompt (or prompt ":"))
  (completing-read prompt list nil t))

;; This is very useful
(defun fz-default-return-query (list &optional prompt)
  (setq prompt (or prompt ":"))
  (completing-read prompt list nil 'confirm))

Prompt using your favorite completion framework

(insert (completing-read "Prompt: " *my-list*))

Helper functions

(defun insert-with-completion (strings)
  "Insert a string from STRINGS into the current buffer.
Prompt for string to insert with completing-read."
  (insert (completing-read "Insert: " strings nil t)))

(defun read-lines (path)
  "Return list of lines in file at PATH."
  (with-temp-buffer
    (insert-file-contents path)
    (split-string (buffer-string) "\n" t)))

Example

(defun insert-from-my-list ()
  "Insert a line into the current buffer, with completion."
  (interactive)
  (insert-with-completion (read-lines "~/path/to/my-list.txt")))

  • Cache it

    (defvar *my-list* nil)

(defun refresh-my-list ()
  "Reread list of things."
  (interactive)
  (setq *my-list* (read-lines "~/path/to/my-list.txt")))

(defun insert-from-my-list-cached ()
  "Insert a line into the current buffer, with completion."
  (interactive)
  (unless *my-list* (refresh-my-list))
  (insert-with-completion *my-list*))
